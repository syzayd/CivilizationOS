## Key Decisions

- **Sustained-tick counter over dice roll**: The old `tick % 45 + 14% random` had no memory — fear could spike and drop repeatedly without ever triggering. Sustained tracking is the minimal correct model for "fear that lingers erupts."
- **Compound threshold 0.78 > base 0.62**: Requiring higher fear for a second auto-crisis during an existing one prevents spam while still allowing cascades in genuine emergencies.
- **300-tick cooldown**: Gives councils time to respond before a second wave fires. At 1 s/tick this is 5 real minutes — aggressive but fair.
- **`kind="emergent"` vs `"crisis"` in event log**: Keeps auto-generated events visually distinct from user-injected ones without adding backend/frontend coupling logic.
- **60-tick post-verdict measurement window**: Immediate fear drop from `_apply_verdict_effects` is mechanical (guaranteed). Measuring 60 ticks later captures whether the narrative + memory effect actually changed citizen behaviour.
- **Effectiveness formula `50 + delta × 150`, centred on 50**: Neutral at 50% so a council that happens to preside over an external shock isn't penalised for fear it didn't cause.
