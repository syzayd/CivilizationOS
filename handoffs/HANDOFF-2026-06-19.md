## What Was Built This Session

### Phase 2 — PANTHEON Councils + TCMF RAG
- `api/memory/causal_graph.py` — NetworkX DiGraph. Nodes = societal events. Edges = causal
  links (A caused B). BFS predecessor traversal with depth limit.
- `api/memory/tcmf.py` — **TCMF retriever**: fuses citizen episodic memory scores with causal
  graph depth boosts. Formula: `tcmf_score = episodic_score × (1 + λ × causal_boost)`.
  Memories near causal ancestors of the current crisis rank higher.
- `api/agents/council.py` — 5-role council debate streaming as AsyncGenerator:
  Historian (📜) → Strategist (⚔️) → Skeptic (🔍) → Predictor (🔮) → Synthesizer (⚖️).
  Synthesizer issues `VERDICT:` directive. Tier.PREMIUM (Claude) for Synthesizer, FREE for others.
- `api/sim/crisis.py` — Crisis registry + debate transcript store.
- `web/src/panels/CouncilChamber.tsx` — live debate UI, crisis injection form.
- `POST /crisis`, `GET /debates/{id}`, `GET /crises` endpoints.

### Phase 3 — Crises + Society Dynamics
- `api/sim/events.py` — 5 crisis templates:
  - 🦠 **Pandemic** → closes clinic, high fear, triggers Healthcare + Government
  - 🌵 **Drought** → closes market, triggers Economy + Government
  - 💻 **Cyberattack** → closes City Hall, triggers Government + Media + Police
  - 🗳️ **Election** → contested results, triggers Government + Media
  - 🔫 **Crime Wave** → closes park, triggers Police + Media
- Citizens gain `fear` (0–1) on crisis injection, decays at 0.006/tick (~2 days to clear).
- Citizens with fear > 0.7 shelter at home regardless of time of day.
- Location closure reroutes citizens to alternative destinations.
- Secondary institutions activate in parallel background debates.
- `web/src/panels/RelationshipGraph.tsx` — canvas social graph (fear heatmap, co-location edges).
- `GET /events/templates` endpoint for frontend presets.

### Phase 4 — Fine-Tuning + MLOps
- `ml/dataset/generate.py` — generates 300-sample instruction-tuning dataset
  (5 roles × 60 crisis scenarios). Run: `python -m ml.dataset.generate --n 60`
- `ml/dataset/council_voices.jsonl` — 300 pre-generated training samples (committed).
- `ml/train_lora.ipynb` — full Unsloth LoRA training notebook for free Colab T4:
  loads Qwen2.5 3B at 4-bit, trains with SFTTrainer, runs persona-consistency eval
  gate, exports GGUF (Q4_K_M) if gate passes. MLflow tracking built in.
- `ml/evals/persona_eval.py` — standalone eval harness: persona-consistency
  (role-specific keyword patterns) + debate-coherence (Jaccard word overlap).
  Logs to MLflow. Promotion gate: overall_pass_rate ≥ 70%.
- `api/config.py` — `OLLAMA_COUNCIL_MODEL` setting (empty = use base model).
- `api/llm/router.py` — `local_model` override parameter on `complete()`.
- Council deliberation automatically uses fine-tuned model when configured.

## How to Run

```powershell
# Backend (from repo root)
$env:PYTHONIOENCODING="utf-8"
.venv\Scripts\python -m uvicorn api.main:app --reload --port 8000

# Frontend
cd web; npm run dev   # http://localhost:5173

# Tests
.venv\Scripts\python -m pytest

# Generate more training data
.venv\Scripts\python -m ml.dataset.generate --n 100 --out ml/dataset/council_voices.jsonl

# Run persona eval (requires Ollama running)
.venv\Scripts\python -m ml.evals.persona_eval --model qwen2.5:3b-instruct --n 5
```

7. Restart backend — councils now use the fine-tuned voice.
