## What this session did

A research effort to turn TCMF (Temporal-Causal Memory Fusion) into a publishable paper, plus
the four `TCMFRetriever` code fixes it surfaced.

1. **Built a benchmark** (`research/tcmf_paper/tcmfbench/`) that evaluates the REAL
   `api/memory/tcmf.py` retriever against scenarios with known causal ground truth. Three tiers
   (synthetic pure, synthetic mixed, real-text via Ollama nomic-embed-text), six baselines, plus
   additive / RRF / multiplicative fusion variants; metrics recall@k, root-cause MRR/rank, nDCG.
2. **Found the core result:** the causal signal is strong (causal-only recall@5 = 1.00) but the
   ORIGINAL multiplicative fusion exploited none of it (recall@5 = 0.00). Normalized ADDITIVE
   fusion of the identical scores recovers it; in the mixed regime it strictly beats every
   single-signal baseline at recall@10 and degrades gracefully under edge dropout; the effect
   survives real embeddings.
3. **Applied four fixes to `api/memory/tcmf.py`:** multiplicative -> additive fusion; exclude
   the crisis node from its own ancestor set; flip the depth weight to favor the root cause;
   remove the pre-fusion per-citizen top-8 prune. Default `causal_boost` raised 0.6 -> 2.0.
